# Module 10: 종합 실전 프로젝트

> 공식 문서: https://code.claude.com/docs/ko/features-overview

## WHY

{name} {title}님, 모듈 1부터 9까지 배운 기능들을 개별적으로는 사용할 수 있게 되었습니다.

하지만 실제 {team}의 업무는 **하나의 기능만으로 끝나는 일이 없습니다.** 고객사 교육 제안서 하나를 만들어도 파일을 읽고(M2), 규칙을 적용하고(M1), 외부 정보를 가져오고(M3), 병렬로 작업을 나누고(M4), 스킬을 활용하고(M5), 팀에 공유하는(M9) 과정이 연결됩니다.

이번 모듈은 **기업교육 컨설턴트로서 본인의 실제 업무 1건을 선정하고, 지금까지 배운 Claude Code 기능을 조합해서 End-to-End로 자동화**하는 종합 프로젝트입니다.

**Before:** 각 기능을 따로따로 사용 — "이걸 언제 조합해서 쓰지?"
**After:** 실제 업무 하나를 통째로 자동화 — "이렇게 연결하면 되는구나!"

## EXPLAIN

### 종합 프로젝트란?

지금까지 배운 Claude Code 기능들을 **실제 업무 1건에 통합 적용**하는 실전 프로젝트입니다.

| 항목 | 내용 |
|------|------|
| 비유 | **교육 컨설턴트의 졸업 프로젝트** — 개별 스킬을 조합해서 실제 고객 프로젝트를 수행하는 것 |
| 목표 | 본인 업무 1건을 Claude Code로 E2E 자동화 |
| 평가 | 퀴즈 없음 — 프로젝트 완성 자체가 평가 |

### 프로젝트 예시

#### 예시 A: 채용 파이프라인 자동화

이력서 분석(파일 도구) → 직무 매칭(Subagent 병렬) → 면접 질문 생성(CLAUDE.md 규칙) → 결과 공유(MCP/Slack)

| 단계 | 활용 기능 | 관련 모듈 |
|------|----------|----------|
| 이력서 파일 읽기 및 정보 추출 | Read/Glob/Grep | M2 |
| CLAUDE.md에 채용 기준 규칙 설정 | CLAUDE.md | M1 |
| 복수 이력서 병렬 분석 | Subagent | M4 |
| 직무별 면접 질문 자동 생성 | 스킬 활용 | M5 |
| 결과를 Slack/Notion에 공유 | MCP | M3 |
| 전체 파이프라인을 팀에 배포 | Agent Teams | M8 |

#### 예시 B: 온보딩 자동화

입사자 정보 수집(MCP) → 체크리스트 생성(파일 도구) → Slack 안내(MCP) → 진행 추적(스킬)

| 단계 | 활용 기능 | 관련 모듈 |
|------|----------|----------|
| CLAUDE.md에 온보딩 규칙 설정 | CLAUDE.md | M1 |
| 입사자 정보 Notion에서 수집 | MCP | M3 |
| 온보딩 체크리스트 파일 생성 | Read/Write | M2 |
| Slack 채널에 환영 메시지 발송 | MCP | M3 |
| 진행 상황 추적 스킬 활용 | 스킬 사용 | M5 |
| 온보딩 스킬을 직접 제작 | Skill Creator | M7 |

#### 예시 C: 본인 업무 자유 선택 ← 권장

{name} {title}님이 {team}에서 **실제로 반복하는 업무** 중 하나를 골라서 자동화합니다.

> 💡 예시 A, B는 참고용입니다. **예시 C를 강력히 추천**합니다 — 본인이 매일 하는 업무를 자동화해야 진짜 체감이 됩니다.

### 4단계 프로세스

모든 프로젝트는 아래 4단계로 진행합니다:

```
선정 → 분해 → 구현 → 테스트
```

| 단계 | 설명 |
|------|------|
| **선정** | 자동화할 업무 1건 선택. 반복성이 높고 Claude Code로 처리 가능한 것 |
| **분해** | 업무를 세부 단계로 쪼개고, 각 단계에 어떤 모듈/기능을 쓸지 매핑 |
| **구현** | 각 단계를 Claude Code로 실제 구현. 파일 생성, MCP 연동, 스킬 활용 등 |
| **테스트** | 전체 흐름을 처음부터 끝까지 실행하여 검증. 개선점 도출 |

## EXECUTE

아래 순서대로 실행해보세요:

---

### 1단계: 프로젝트 선정

📋 아래를 복사해서 입력하세요:

    나는 {division} {team}에서 기업교육 컨설턴트로 일하고 있어.
    내가 반복적으로 하는 업무 중에서 Claude Code로 자동화하기 좋은 것을 3가지 추천해줘.
    각각 어떤 모듈의 기능을 조합하면 되는지도 알려줘.

추천 목록 중 하나를 선택하거나, 본인이 직접 정한 업무를 말씀하세요.

---

### 2단계: 업무 분해

선택한 업무를 단계별로 쪼갭니다:

📋 아래를 복사해서 입력하세요:

    내가 선택한 업무는 [선택한 업무]야.
    이 업무를 세부 단계로 분해하고, 각 단계에서 쓸 Claude Code 기능을 매핑해줘.
    표로 정리해줘: 단계 | 작업 내용 | 활용 기능 | 관련 모듈

---

### 3단계: 구현

분해한 각 단계를 실제로 구현합니다:

📋 아래를 복사해서 입력하세요:

    분해한 단계를 하나씩 구현해보자.
    1단계부터 시작해줘. 필요한 파일 생성, MCP 연동, CLAUDE.md 규칙 등을 세팅해줘.

> 💡 한 번에 전부 구현하려 하지 마세요. 단계별로 하나씩 확인하면서 진행하는 것이 좋습니다.

---

### 4단계: 테스트 및 개선

전체 흐름을 처음부터 끝까지 실행합니다:

📋 아래를 복사해서 입력하세요:

    전체 자동화 흐름을 처음부터 끝까지 테스트해줘.
    각 단계가 제대로 연결되는지 확인하고, 개선할 점이 있으면 알려줘.
