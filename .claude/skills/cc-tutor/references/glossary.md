# HR 채용 담당자 용어 비유 사전

> SKILL.md가 Phase A에서 기술 용어를 설명할 때 이 사전을 참조한다.
> 비유는 HR 채용 담당자의 일상 업무에서 가져온다.

## 용어 비유 테이블

| 기술 용어 | HR 채용 비유 | 한 줄 설명 |
|----------|-------------|-----------|
| CLI (터미널) | 나인하이어에 직접 명령 | 마우스 대신 텍스트로 컴퓨터에 직접 지시하는 방식. 나인하이어에서 필터를 직접 걸듯, 명령어로 바로 실행한다 |
| CLAUDE.md | 채용 SOP 가이드 | Claude가 매 세션 시작 시 읽는 규칙서. "JD는 항상 필수 요건과 우대 요건을 분리" 같은 규칙을 저장한다 |
| Auto Memory | 채용 히스토리 노트 | Claude가 작업하면서 스스로 적는 메모. 이전 채용 프로세스 인사이트를 기억하고 있다 |
| Skill | 검증된 채용 템플릿 풀 | 특정 업무에 특화된 전문 역량. 필요할 때 "/스킬명"으로 바로 꺼내 쓴다 |
| MCP | 채용 플랫폼 연동 (잡포털, LinkedIn 등) | Claude가 외부 시스템(Slack, Notion, 잡포털 등)과 실시간으로 연결되는 파이프 |
| Context Sync | 포지션별 채용 현황 대시보드 | 여러 채널에 흩어진 채용 데이터를 한 문서로 모으는 것. 현황 파악할 때 여러 시스템 뒤지는 대신 한 마디로 통합 |
| Subagent | 채용 TF에 동시 브리핑 | {name} {title}님이 여러 포지션의 리서치를 동시에 요청하는 것. AI가 이걸 병렬로 처리한다 |
| Agent Teams | 채용 위원회 | JD 작성자, 소싱 담당, 스크리너 등 역할별 에이전트를 조합한 팀. AGENTS.md가 각 에이전트의 JD 역할을 한다 |
| Plugin | 외부 헤드헌터 투입 | 스킬보다 무겁지만 독립적 기능을 가진 외부 도구. 계약(설치)해서 프로젝트에 투입한다 |
| Hook | JD 자동 품질 체크 | "채용 공고 저장하면 자동으로 포맷·톤 검수" 같은 조건부 자동 실행 규칙 |
| Eval | 채용 프로세스 효과성 검증 | 스킬이 제대로 작동하는지 테스트 시나리오로 검증. 채용 프로세스 개선 전/후를 비교하는 것과 같다 |
| GitHub repo | 채용 에셋 공유 폴더 | 스킬, 설정, 에이전트를 팀원에게 배포하는 저장소. `npx skills add` 한 줄로 설치 가능 |
| Compound | 채용 인사이트 축적 | 작업 중 발견한 인사이트를 구조화해서 축적. "이 JD 표현이 지원율 2배 높았다" 같은 지식을 자동 기록 |
| Session Wrap | 채용 진행 상황 일지 | 세션 종료 시 오늘 한 일, 배운 것, 다음 할 일을 자동으로 정리해주는 기능 |
| Clarify | 채용 요건 인터뷰 | 모호한 요청을 체계적 질문으로 구조화하는 플러그인. 현업 니즈 인터뷰처럼 빠짐없이 확인한다 |
| skill-creator | 팀 SOP를 AI에게 가르치는 것 | 업무 노하우를 SKILL.md로 패키징하는 도구. 신입에게 매뉴얼 만들어주듯, AI에게 노하우를 전수한다 |
| PRD | 포지션 요구사항 정의서 | 무엇을 만들지 정의하는 기획 문서. 포지션 JD가 역할을 정의하듯, PRD가 소프트웨어를 정의한다 |

## 사용법

SKILL.md의 Phase A에서 기술 용어가 처음 등장할 때 이 사전을 참조한다.

### 적용 절차

1. 이 테이블에서 해당 용어의 비유를 찾는다
2. 개인화 변수(`{name}`, `{title}`, `{colleagues}` 등)를 실제 값으로 치환한다
3. 비유와 함께 기술 용어를 설명한다

### 예시

원본 (glossary 템플릿):
> "Subagent는 {name} {title}님이 여러 포지션의 리서치를 동시에 요청하는 것과 같습니다."

치환 후 (실제 출력):
> "Subagent는 연수 담당님이 3개 포지션의 시장 보상 리서치를 동시에 요청하는 것과 같습니다. AI가 이걸 병렬로 처리합니다."

### 주의사항

- 한 모듈에서 같은 용어가 여러 번 나올 때, 비유는 **처음 등장할 때만** 제공한다
- 이후에는 기술 용어를 그대로 사용한다 (반복 비유는 오히려 학습을 방해)
- 학습자가 "도움 → 무슨 말인지 모르겠어요"를 선택하면, 이 사전에서 다시 비유를 제공한다
