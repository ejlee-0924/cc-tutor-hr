## WHY

{name} {title}님, 매주 교육 제안서를 몇 건이나 작성하시나요?

{team}의 컨설턴트들은 고객사 니즈 분석, 강사 리서치, 제안서 작성, 일정 조율을 동시에 처리합니다. Claude Code는 이 모든 업무를 함께할 **새로운 팀원**입니다.

오늘은 이 팀원에게 **첫 브리핑**을 하는 겁니다. 설치하고, 인사하고, 첫 업무를 맡겨봅니다.

**Before:** 고객사 교육 니즈를 정리하려면 메일·슬랙·노션을 뒤져서 수동으로 취합 (30분+)
**After:** "고객사 DT교육 관련 요청사항 정리해줘" 한 마디면 끝

그런데 설치만으로는 부족합니다. {team}에 새 컨설턴트가 들어오면 제일 먼저 뭘 알려주시나요?

제안서 톤앤매너, 고객 응대 원칙, 자주 쓰는 강사 풀, 금액 산정 기준... 이런 팀 규칙을 매번 말로 전달하시죠? **CLAUDE.md는 Claude에게 이걸 한 번 알려주면 영원히 기억하게 하는 것**입니다.

**Before:** 매번 "존댓말로 써줘", "표로 정리해줘", "고객 과제부터 시작해" 반복
**After:** CLAUDE.md에 한 번 저장 → 모든 세션에서 자동 적용

## EXPLAIN

### Part A: Claude Code 기본

#### Claude Code란?

터미널에서 `claude` 한 글자를 치면 시작되는 AI 어시스턴트입니다.

| 항목 | 내용 |
|------|------|
| 비유 | **새로 합류한 주니어 컨설턴트** — 지시하면 리서치, 문서 작성, 파일 정리를 해줍니다 |
| 시작법 | 터미널에서 `claude` 입력 |
| 특징 | 파일을 직접 읽고 쓸 수 있음, 명령어를 실행할 수 있음, 대화하면서 업무를 처리 |

#### 터미널이란?

> 📖 glossary 참조: CLI = "고객사에 전화하기"

마우스로 클릭하는 대신, 텍스트로 컴퓨터에 직접 지시하는 창입니다.
고객사에 전화해서 바로 요점을 전달하듯, 명령어로 바로 실행합니다.

**터미널 여는 법:**
- **Windows:** `Win + R` → `cmd` 입력 → Enter (또는 시작 메뉴에서 "PowerShell" 검색)
- **Mac:** `Cmd + Space` → "터미널" 검색 → Enter

#### 권한 시스템

Claude Code는 파일 읽기/쓰기, 명령어 실행 전에 허락을 구합니다.
새 팀원이 {name} {title}님에게 "이거 해도 돼요?" 확인받는 것과 같습니다.

| 권한 | 설명 |
|------|------|
| Allow once | 이번 한 번만 허락 |
| Allow always | 이 작업은 항상 허락 (자주 하는 작업에 편리) |
| Deny | 거절 |

> 💡 처음에는 **Allow once**로 시작하세요. 뭘 하는지 눈으로 확인하는 습관이 중요합니다.

#### 에디터 (선택사항)

Claude Code는 터미널에서 `claude`만 치면 됩니다. 파일을 눈으로 보면서 작업하고 싶다면:

| 에디터 | 추천 대상 | 사이트 |
|--------|----------|--------|
| Antigravity | 처음 시작하는 분 | antigravity.google |
| Cursor | 코드 편집도 할 분 | cursor.com |
| VSCode | 이미 쓰고 계신 분 | code.visualstudio.com |

> "이 과정에서는 에디터 없이 터미널만으로 충분합니다."

#### 꿀팁: 숨겨진 폴더 보기

`.`으로 시작하는 폴더(`.claude` 등)는 기본적으로 안 보입니다.

**Windows:** 파일 탐색기 → 상단 "보기" → "숨김 항목" 체크
**Mac:** Claude Code에게 `Finder에서 숨겨진 폴더도 볼 수 있게 설정해줘` 라고 요청

설정 후 폴더 열기:
- **Windows:** 탐색기에서 해당 폴더로 이동
- **Mac:** `open ./`

> `.claude/skills/` 폴더 안에 지금 배우고 있는 교안이 들어있습니다!

#### Output Style 설정

Claude Code에는 **응답 스타일**을 바꿀 수 있는 기능이 있습니다. 마치 신입 팀원에게 "짧게 답해" vs "왜 그런지 설명도 해줘"를 지시하는 것과 같습니다.

| 스타일 | 특징 | 추천 상황 |
|--------|------|----------|
| Concise | 짧고 핵심만 | 이미 익숙할 때 |
| **Explanatory** | **왜 그렇게 하는지 설명 포함** | **지금처럼 배우는 단계** |
| Formal | 격식체, 보고서 톤 | 공식 문서 작성 |

`/output-style`을 입력하면 선택 화면이 나옵니다. 학습 중에는 **Explanatory**를 추천합니다 — Claude가 작업하면서 "이렇게 한 이유"를 함께 알려줘서 배우기 좋습니다.

### Part B: CLAUDE.md + Auto Memory

#### 1. CLAUDE.md — 내가 쓰는 지시문

| 항목 | 내용 |
|------|------|
| 비유 | **팀 제안서 가이드라인** — Claude가 매 대화 시작할 때 읽는 규칙서 |
| 근본 원리 | 시스템 프롬프트 — AI가 대화를 시작할 때 가장 먼저 읽는 지시문 |
| 예시 | "항상 존댓말로", "제안서는 고객 과제부터 시작", "금액은 VAT 별도로 표기" |

```
세션 시작
  │
  ▼
┌─────────────┐     ┌─────────────┐
│ CLAUDE.md   │────▶│ 시스템       │────▶ Claude가 규칙을 아는 상태로 대화 시작
│ (내 지시문)  │     │ 프롬프트     │
└─────────────┘     └─────────────┘
  항상 읽힘            매번 자동 주입
```

#### 2. Auto Memory — Claude가 스스로 적는 메모

| 항목 | 내용 |
|------|------|
| 비유 | **Claude의 업무 수첩** — 일하면서 적어두는 자기 노트 |
| CLAUDE.md와 차이 | CLAUDE.md는 **내가** Claude에게 주는 규칙. Auto Memory는 **Claude가** 스스로 적는 메모 |
| 저장 위치 | `~/.claude/projects/<프로젝트>/memory/` |

**Auto Memory가 기록하는 것들:**
- 프로젝트 패턴: 자주 쓰는 파일 구조, 명명 규칙
- 디버깅 인사이트: 까다로운 문제의 해결법
- 내 선호사항: 소통 방식, 워크플로우 습관

#### 3. 계층 구조 — 용도에 따라 나눠서 쓰기

CLAUDE.md는 여러 레벨에 둘 수 있고, **각 레벨마다 넣을 내용이 다릅니다:**

| 레벨 | 위치 | 넣을 내용 | 비유 |
|------|------|----------|------|
| **나의 기본** | `~/.claude/CLAUDE.md` | 나에 대한 정보, 소통 선호 | **내 명함 + 업무 스타일** |
| **프로젝트** | 프로젝트 폴더의 `CLAUDE.md` | 이 프로젝트의 업무 규칙 | **프로젝트별 가이드라인** |

> 💡 **나의 기본**은 어떤 프로젝트에서든 항상 적용됩니다. **프로젝트**는 해당 폴더에서만 적용됩니다.

## EXECUTE

### Part A: 작업환경 세팅

아래 순서대로 실행해보세요:

**1단계: Claude Code 실행**

터미널을 열고 아래 명령어를 입력합니다:

    claude

로그인 화면이 나오면 Anthropic 계정으로 로그인하세요.

**2단계: 첫 인사**

Claude Code에 이렇게 말해보세요:

    안녕, 나는 {name}이고 {division} {team}에서 기업교육 컨설턴트로 일하고 있어. 나한테 인사해줘.

**3단계: 첫 업무 맡기기**

실제 업무와 관련된 요청을 해보세요:

    고객사에서 "AI 기반 업무 자동화" 주제로 임직원 교육을 요청했어.
    이 교육의 니즈를 분석하고 주요 교육 포인트를 5가지로 정리해줘.

**4단계: Output Style 설정**

아래 명령어를 입력하면 스타일 목록이 나옵니다:

    /output-style

목록에서 **Explanatory**를 선택하세요. Claude가 "왜 이렇게 했는지"를 함께 설명해주는 모드입니다. 나중에 익숙해지면 Concise로 바꿔도 됩니다.

### Part B: CLAUDE.md 작성

**실습 1: 나의 기본 CLAUDE.md 만들기 (나에 대한 정보)**

먼저 **나는 누구인지** AI에게 알려줍니다. 이 파일은 어떤 프로젝트에서든 항상 적용됩니다.

Claude Code에 이렇게 요청하세요:

    ~/.claude/CLAUDE.md 파일에 다음 내용을 넣어줘:

    # 나에 대한 정보
    - 이름: {name}
    - 소속: {division} {team}
    - 역할: 기업교육 컨설턴트

    # 소통 선호
    - 항상 한국어로 대화
    - 설명할 때 비유와 예시를 함께 사용
    - 비개발자 눈높이에 맞춰 설명

> 이제 Claude는 **어떤 프로젝트에서든** {name} {title}님이 누구인지 알고 대화를 시작합니다.

**실습 2: 프로젝트 CLAUDE.md 만들기 (제안서 규칙)**

> 💡 모듈 2에서 만든 RFP 기반 제안서가 proposals 폴더에 있죠? 이제 그 제안서를 **일관된 규칙**으로 작성하도록 세팅합니다.

이번에는 **이 프로젝트의 규칙**을 설정합니다. `/init` 명령어로 시작하면 편합니다:

    /init

> ⚠️ **주의:** 이미 CLAUDE.md가 있는 프로젝트에서 `/init`을 실행하면 기존 내용이 덮어씌워질 수 있습니다. 기존 파일이 있다면 `/init` 대신 직접 수정하세요.

자동 생성된 CLAUDE.md에 컨설팅 규칙을 추가합니다:

    CLAUDE.md에 다음 규칙들을 추가해줘:

    ## 제안서 작성 규칙
    - 항상 존댓말로 작성
    - 제안서 구조: 고객 과제 → 교육 목표 → 커리큘럼 → 강사진 → 일정/예산 → 기대 효과
    - 금액은 VAT 별도로 표기
    - 고객사명은 정식 명칭 사용

    ## 강사 분류 체계
    - AI/DT: AI 전문가, DT 컨설턴트
    - 리더십: 리더십 코치, 조직 개발 전문가
    - 직무: 직무별 현업 실무자

규칙을 저장한 후, 모듈 2에서 만든 제안서 초안을 다시 읽게 해보세요:

    proposals 폴더의 제안서 초안을 읽고, 방금 설정한 규칙에 맞게 수정해줘.

> CLAUDE.md 규칙이 적용되면서 제안서 톤과 구조가 달라지는 걸 확인해보세요!

**실습 3: /memory로 확인**

    /memory

나의 기본 CLAUDE.md와 프로젝트 CLAUDE.md가 각각 잘 설정되었는지 확인해보세요.

**실습 4: Auto Memory 직접 체험**

Claude Code에 이렇게 말해보세요:

    나는 제안서 쓸 때 항상 고객의 비즈니스 임팩트를 먼저 강조하는 걸 좋아해. 이걸 기억해둬.

`/memory`로 저장된 내용을 확인해보세요. 실습 1에서 넣은 **개인 정보**와 달리, 이건 **Claude가 스스로 메모**한 것입니다.

## QUIZ

```json
AskUserQuestion({
  "questions": [{
    "question": "CLAUDE.md와 Auto Memory의 차이는 무엇인가요?",
    "header": "Quiz 1-1",
    "options": [
      {"label": "CLAUDE.md는 내가 쓰고, Auto Memory는 Claude가 쓴다", "description": "각각 작성 주체가 다르다"},
      {"label": "같은 파일의 다른 이름이다", "description": "이름만 다른 같은 기능?"},
      {"label": "Auto Memory는 한 번만 읽힌다", "description": "매 세션마다 읽히는지?"}
    ],
    "multiSelect": false
  }]
})
```

정답: 1번. CLAUDE.md는 **내가 Claude에게 주는 지시문**(제안서 가이드라인)이고, Auto Memory는 **Claude가 작업하면서 스스로 기록하는 메모**(업무 수첩)입니다. 둘 다 매 세션 시작 시 자동으로 읽힙니다.

```json
AskUserQuestion({
  "questions": [{
    "question": "제안서를 항상 '고객 과제 → 교육 목표 → 커리큘럼' 순서로 쓰게 하려면 어디에 규칙을 넣나요?",
    "header": "Quiz 1-2",
    "options": [
      {"label": "CLAUDE.md", "description": "내가 Claude에게 주는 규칙서"},
      {"label": "Auto Memory", "description": "Claude가 스스로 적는 메모"},
      {"label": "매번 대화로 알려준다", "description": "반복 지시"}
    ],
    "multiSelect": false
  }]
})
```

정답: 1번. 팀 규칙처럼 항상 적용되어야 하는 것은 CLAUDE.md에 넣습니다. 매번 말할 필요가 없어집니다.
