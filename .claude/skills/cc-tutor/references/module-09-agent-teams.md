# Module 9: Agent Teams

> 공식 문서: https://code.claude.com/docs/ko/agent-teams

## WHY

{name} {title}님, 채용 공고 하나 완성하려면 시장조사도 하고, 직무 요건도 짜고, 검수도 해야 하죠. 혼자서 이걸 다 하시잖아요. **Agent Teams는 이 과정을 AI TF로 구성해서 한 번에 돌리는 것**입니다.

**Before:** JD 작성 → 검수 → 수정 → 재검수를 혼자 반복 (3일)
**After:** AI TF가 리서치→초안→검수까지 파이프라인으로 실행 (3시간)

## EXPLAIN

### Agent Teams란?

> 📖 glossary 참조: Agent Teams = "프로젝트 TF"

여러 AI 에이전트에게 각각 다른 역할을 부여하고, 하나의 프로젝트를 협업으로 완성하게 하는 것입니다.

```
{name} {title}님 (프로젝트 오너)
    │
    ▼
AGENTS.md (각 에이전트의 JD)
    │
    ├──▶ 시장조사 에이전트 ──▶ 산업 동향 + 경쟁사 분석
    ├──▶ JD 작성 에이전트 ──▶ 리서치 기반 JD 초안
    └──▶ QA 에이전트 ──▶ 논리 흐름 + 누락 항목 + 톤 검수
    │
    ▼
검증된 제안서 완성
```

### AGENTS.md — 에이전트의 JD

| 항목 | 설명 |
|------|------|
| 비유 | 각 TF 멤버의 직무기술서(JD) |
| 역할 | 에이전트 이름, 전문 분야, 접근 권한, 업무 범위 |
| 위치 | 프로젝트 폴더의 `.claude/` 안에 작성 |

```markdown
# AGENTS.md 예시

## researcher
- 역할: 시장조사 및 경쟁사 분석
- 전문: 산업 동향, 교육 트렌드, 후보자 소싱
- 접근: 웹 검색, 문서 읽기

## writer
- 역할: 채용 공고 작성
- 전문: JD 구조화, 톤앤매너, 직무 요건 설계
- 접근: 리서치 결과, 템플릿, CLAUDE.md 규칙

## reviewer
- 역할: 제안서 품질 검수
- 전문: 논리 검증, 누락 체크, 지원자 관점 검토
- 접근: 최종 제안서, 현업 요구사항 원본
```

### 에이전트 간 협업 방식

| 방식 | 설명 | 적합한 상황 |
|------|------|-----------|
| 순차형 | A→B→C 순서대로 | 채용 파이프라인 (리서치→작성→검수) |
| 병렬형 | A, B, C 동시에 | 독립적 리서치 3건 동시 진행 |
| 대립형 | A vs B → C가 종합 | 의사결정 검증 (찬성 vs 반대 → 판정) |

### 실습 9-1: 채용 파이프라인 TF 구성 [목표 A]

📋 아래를 복사해서 입력하세요:

    개발팀에서 백엔드 엔지니어 채용을 요청했어.

    Agent Teams로 3명의 TF를 구성해서 JD를 만들어줘:

    1. 시장조사 에이전트
       - 반도체 산업의 AI 활용 동향
       - 경쟁사 교육 프로그램 분석
       - 최신 AI 교육 트렌드

    2. JD 작성 에이전트
       - 리서치 결과를 바탕으로 JD 초안
       - 우리 팀 CLAUDE.md 규칙 따르기
       - 6섹션 구조

    3. QA 에이전트
       - 논리 흐름 검증
       - 누락 항목 체크
       - 톤앤매너 일관성 확인

    먼저 AGENTS.md를 작성하고, 실제로 End-to-End 실행해줘.

Agent Teams가 순차적으로 작업하는 과정을 관찰하세요. 리서치 → 초안 → 검수 흐름이 자동으로 진행됩니다.

### 실습 9-2: 의사결정 검증 [목표 B]

9-1에서 만든 JD를 3관점으로 검증합니다:

📋 아래를 복사해서 입력하세요:

    방금 만든 SK하이닉스 JD를 3개 관점으로 검증해줘:

    1. 찬성 에이전트 — 이 JD의 강점과 설득 포인트를 정리
    2. 반대 에이전트 — 약점, 리스크, 현업이 반론할 수 있는 포인트를 지적
    3. 지원자 관점 에이전트 — SK하이닉스 교육 담당자 입장에서 평가

    3개 관점을 종합해서, JD를 최종 보강해줘.

현업에 JD를 공유하기 전에, AI로 미리 "품평회"를 여는 겁니다.

---

### 실습 9-3: 확장 설계 [리더 실습]

📋 아래를 복사해서 입력하세요:

    이 채용 파이프라인 + 의사결정 검증 구조를,
    분기별 채용 공고 10건을 동시에 처리할 수 있는 구조로 확장 설계해줘.

    각 제안서마다 3-에이전트 TF가 독립적으로 작동하고,
    전체를 관리하는 매니저 에이전트가 있는 구조로.

## LEADER-TIP

> 💡 **{name} {title}님을 위한 리더 팁**
>
> - 이 구조를 **정규 워크플로우**로 만들면, 모든 JD가 3중 검증을 거치게 됩니다
> - 혼자 하는 업무도 AI TF를 쓰면 **여러 관점에서 검증된 결과물**을 낼 수 있습니다
> - **두 가지 활용 패턴을 업무에 적용하세요:**
>   - 📋 **채용 파이프라인** (리서치→작성→검수) — 모든 JD에 적용
>   - ⚖️ **의사결정 검증** (찬성 vs 반대 vs 지원자 관점) — 중요 JD에 적용
> - 현업에 JD를 공유하기 전 "AI 품평회"를 거치세요 — 혼자서도 다각도 검증 가능
> - 🤝 **AI 설계자 배지를 획득했습니다!** 이제 AI TF를 설계하고 운영할 수 있습니다
